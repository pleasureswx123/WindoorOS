# WindoorOS 视觉风格与交互设计方向

版本：v1.0  
日期：2026-05-27

## 1. 推荐风格

WindoorOS 推荐采用“飞书式国内 SaaS 工作台风格”作为主视觉方向。

参考来源：

- 主参考：飞书 Feishu 风格
- 辅助参考：Linear 的专业工具台信息组织
- 不建议作为主参考：Notion、汽车品牌、强营销型落地页

## 2. 为什么选飞书风

WindoorOS 的核心用户是门窗小老板和师傅，系统需要长期、高频、在现场和店里都能用。

飞书风适合本产品的原因：

- 国内用户熟悉，学习成本低。
- 白底为主，适合户外手机查看。
- 蓝色主操作明确，适合表单、流程、保存、导出等业务动作。
- 信息密度适中，适合客户、订单、尺寸、材料、报价等复杂数据。
- 视觉可信赖，不会显得像玩具或纯营销页。

Linear 风格的专业感可以借鉴，但不建议全站深色：

- 深色适合开发工具、监控台、专业任务面板。
- 门窗现场量尺常在白天户外，深色对可读性不一定友好。
- 老板和师傅更需要“看得清、点得准、少出错”。

## 3. 设计关键词

```text
清晰、可信、效率、工程感、少装饰、强数据、好打印、好导出
```

禁止方向：

- 过度炫酷
- 过度渐变
- 大面积深色
- 营销页式大卡片堆叠
- 只好看但信息密度低
- 把画布做成普通涂鸦板

## 4. 视觉基调

### 4.1 色彩

主色：

```text
主蓝：#3370FF
主蓝 Hover：#2A5FE6
主蓝浅底：#F0F5FF
```

中性色：

```text
文字主色：#1F2329
正文文字：#51565E
弱文字：#8A9099
边框：#DCDFe5
浅边框：#EBEDF0
页面背景：#F7F8FA
卡片背景：#FFFFFF
```

业务色：

```text
成功：#00BA88
警告：#FF9226
危险：#F53F3F
信息：#3370FF
```

行业辅助色：

```text
型材棕灰：#8A6A52
玻璃浅蓝：#DFF6FA
尺寸标注深灰：#2B3038
余料警告橙：#FF9226
```

### 4.2 圆角

遵循工具型产品，不使用过大圆角。

```text
小控件：6px
按钮/输入框：8px
面板：8px
弹窗：12px
```

### 4.3 字体

```text
中文：PingFang SC / Microsoft YaHei / Noto Sans SC
英文与数字：Inter / system-ui
代码与尺寸：JetBrains Mono / ui-monospace
```

尺寸数字建议使用等宽或 tabular figures，避免报价、尺寸表跳动。

## 5. 页面结构

## 5.1 未登录宣传体验页

目标：

- 说明价值，但不阻止用户体验。
- 第一屏直接看到产品能力，不做空洞品牌页。

第一屏结构：

```text
顶部导航：Logo / 功能 / 价格 / 登录
左侧：一句话价值 + 两个按钮
右侧：真实产品界面预览
下方露出：画窗、算料、报价三个功能区
```

主标题建议：

```text
门窗量尺、算料、报价，一次做完
```

副标题：

```text
手机现场记录尺寸，电脑回店生成切割方案和报价单。
```

按钮：

- 主按钮：开始画窗
- 次按钮：查看示例订单

## 5.2 登录后工作台

推荐布局：

```text
顶部栏：当前门店 / 搜索 / 新建 / 用户
左侧栏：客户、订单、量尺、材料、报价、生产
主区域：当前订单工作区
右侧：当前窗户参数/报价摘要
```

工作台不使用营销式 hero，不用大面积装饰图。

## 5.3 订单工作区

核心布局：

```text
左侧：窗户列表
中间：门窗画布
右侧：尺寸与结构参数
底部/抽屉：材料、切割、报价结果
```

手机端：

```text
顶部：客户与订单
中间：画布
底部：分段标签，窗户 / 参数 / 算料 / 报价
```

## 6. 关键组件规范

### 6.1 按钮

主按钮：

- 蓝底白字
- 8px 圆角
- 用于保存、生成报价、导出、登录

次按钮：

- 白底灰边框
- 用于取消、复制、查看

危险按钮：

- 红色文字或红色边框
- 用于删除客户、删除窗户

### 6.2 表单

要求：

- 所有尺寸字段明确显示 `mm`。
- 手机端数字输入必须调起数字键盘。
- 常用值提供快捷按钮，例如 3mm/5mm 锯缝。
- 表单错误必须就地提示。

### 6.3 画布

画布是产品核心，不做成普通白板。

要求：

- 外框、中梃、玻璃、开启方向都有稳定视觉符号。
- 尺寸标注必须清楚。
- 拖拽点要大，适合手指。
- 选中对象要有高亮边框。
- 支持放大、缩小、适应屏幕。
- 手机端避免细小难点的控件。

视觉建议：

```text
外框：型材棕灰或深灰
玻璃：浅蓝渐变或淡蓝底
中梃：比外框略深
尺寸线：深灰
选中态：飞书蓝
错误态：红色
```

### 6.4 数据表

用于：

- 客户列表
- 订单列表
- 材料清单
- 下料清单
- 报价明细

要求：

- 表头固定，行高 44-52px。
- 金额和尺寸右对齐。
- 状态使用小标签。
- 支持手机端转为列表卡片。

### 6.5 状态标签

颜色建议：

```text
待量尺：灰色
待报价：蓝色
已报价：紫色或蓝色
已成交：绿色
生产中：橙色
已安装：绿色
已流失：灰色
异常/尺寸冲突：红色
```

## 8. 移动端交互

### 8.1 现场量尺模式

手机端要优先服务现场场景：

- 新建客户一键进入量尺。
- 添加窗户只要 3 步：位置、宽高、结构。
- 支持快速复制。
- 支持拍现场照片。
- 支持语音备注，后续版本。

### 8.2 手势

支持：

- 双指缩放画布
- 单指拖动中梃
- 长按选中构件
- 点击尺寸标注进入数字编辑

避免：

- 过多隐藏手势
- 小于 44px 的可点击目标
- 依赖鼠标悬停才能发现功能

## 9. PC 端交互

电脑端强调效率：

- 快捷键，后续版本
- 批量复制窗户
- 表格快速录入
- 右侧参数面板实时更新
- 打印报价单
- 查看完整切割方案

## 10. 信息密度

WindoorOS 是生产工具，信息密度应高于普通官网，低于传统 ERP。

原则：

- 首屏显示当前订单关键信息。
- 画布永远是工作区中心。
- 报价和切割结果可折叠。
- 不用大面积介绍文字占据工作空间。

## 11. 设计验收标准

视觉验收：

- 第一眼能看出是门窗行业工具。
- 手机和电脑都能清楚看到尺寸。
- 页面主操作明确。
- 画布与参数表不会互相抢空间。
- 报价单适合客户阅读。
- 下料单适合师傅照着切。

交互验收：

- 现场 30 秒内能新增一个窗户。
- 复制一个相同窗户不超过 2 次点击。
- 导出效果图不超过 2 次点击。
- 生成报价不超过 3 次点击。

## 12. 推荐最终结论

主风格采用：

```text
飞书式浅色企业工作台
```

局部借鉴：

```text
Linear 式专业数据面板和状态层级
```

不采用：

```text
全站深色、强营销渐变、大圆角卡片堆叠、Notion 式轻文档风
```

这个方向最符合门窗老板的真实使用环境：现场看得清，回店算得准，师傅拿到方案能直接干活。
